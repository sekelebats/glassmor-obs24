  • Memulai Blog Minimalis: Fokus pada Tulisan 24 May 2026

    Blog ini lahir dari keinginan sederhana: tempat menulis tanpa gangguan. Tidak ada iklan, tidak ada metrik yang bikin pusing, hanya ruang kosong yang siap diisi kata-kata. Saya percaya bahwa semakin sedikit elemen visual yang bersaing, semakin kuat pesan yang tersampaikan.

  • Bermain dengan Format Tulisan: Contoh Nyata 17 May 2026

    Salah satu kelebihan menulis dengan HTML adalah kita bisa mengatur sendiri struktur teks. Di bawah ini beberapa contoh elemen yang sering saya pakai:

    • Daftar tak berurutan: untuk ide yang setara.
    • Daftar berurutan: untuk langkah yang harus diikuti.
    • Kode dalam baris: misalnya, perintah npm install atau variabel const x = 10;.

    Kemudian untuk kutipan, saya sering menyisipkan blockquote seperti ini:

    "Menulis adalah cara paling sederhana untuk mengabadikan pikiran. Tidak perlu menunggu sempurna, tulis saja dulu."

    Adapun blok kode panjang sangat berguna untuk menampilkan potongan skrip:

    function sapa(nama) {
  return `Halo, ${nama}! Selamat datang di blog ini.`;
}
console.log(sapa('FZH'));

    Dengan kombinasi di atas, artikel teknis atau cerita pribadi jadi lebih hidup. Kamu bisa langsung mempraktikkannya di editor HTML favoritmu.

  • Cara Menulis Blog: HTML Mentah, SSG, atau Flat-File CMS? 26 Apr 2026

    Ada banyak cara mengelola konten blog. Berikut beberapa pendekatan yang bisa kamu pilih:

    1. HTML mentah (raw): Tulis langsung di file HTML. Cocok untuk blog sederhana tanpa banyak halaman. Keuntungan: tidak butuh build tool, langsung tayang. Kekurangan: harus update manual.
    2. Static Site Generator (SSG): Contoh: Eleventy, Hugo, atau Astro. Kamu menulis dalam Markdown, lalu build menghasilkan HTML statis. Cocok untuk blog dengan banyak artikel dan ingin otomatisasi.
    3. Flat-File CMS: Misalnya Grav atau Bludit. Tidak perlu database, konten disimpan dalam file teks. Praktis untuk pengguna yang ingin antarmuka admin tanpa kerumitan database.
    4. Headless CMS + SSG: Kombinasi modern: konten dikelola di CMS (seperti Decap CMS atau Strapi) lalu di-fetch saat build. Cocok untuk tim.

    Blog ini sendiri menggunakan HTML murni, karena saya lebih suka kendali penuh. Setiap artikel adalah file HTML terpisah yang diunggah ke hosting. Untuk kamu yang baru memulai, coba dulu HTML mentah. Jika sudah bosan, naikkan level ke SSG. Tidak ada aturan baku, yang penting nyaman dan konsisten menulis.

  • Dari GitHub ke Netlify: Alur Hosting Modern 19 Apr 2026

    Kode blog ini disimpan di repositori GitHub. Setiap perubahan yang di-push ke branch utama langsung memicu proses build di Netlify. Hasilnya, situs diperbarui otomatis dalam hitungan detik.

    Mengapa Netlify? Karena mereka menyediakan CDN global, SSL gratis, dan integrasi mulus dengan GitHub. Pengunjung dari mana pun akan merasakan waktu muat yang cepat berkat distribusi konten ke server terdekat.

    Untuk blog sekecil ini, paket gratis Netlify sudah lebih dari cukup. Bahkan fitur deploy preview sangat membantu saat ingin melihat perubahan sebelum tayang publik. Praktis, cepat, dan tanpa biaya.
